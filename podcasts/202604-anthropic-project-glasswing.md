# Podcast 大綱：Anthropic Project Glasswing 與 Claude Mythos Preview

**預計長度：約 10 分鐘**
**形式：重點大綱與數據整理，供主持人自由發揮語氣。**

---

### [Part 1] 開場：破天荒的不發布宣言（約 2 分鐘）
- **重點新聞**：Anthropic 宣布了一個全新的 AI 模型「Claude Mythos Preview」。
- **反常舉動**：這模型並非專為找漏洞而生，但因為他們發現它抓漏實在太強太危險，因此宣布「不對大眾公開發售」，僅限內部與指定防禦合作夥伴使用。
- **原因**：資安攻防能力過於駭人，直接釋出風險極高，是業界首次主動踩煞車的案例。
- **引用 Boris Cherny (Claude Code 創辦人) 說法**：
  - 「Mythos 非常強大，強大到應該讓人感到害怕。我很驕傲我們選擇了負責任的方式，先讓資安防禦者使用，而不是直接丟到市場上。」

### [Part 2] 數據說話：Mythos 的驚人成績（約 3 分鐘）
- **標準跑分對比**：
  - **SWE-bench Verified** (寫 code 測試)：Mythos **93.9%** vs. 前代 Opus 4.6 (80.8%)
  - **Cybench** (35 道 CTF 挑戰題)：Mythos **100% 通過率**（全數解開）
  - **CyberGym** (真實開源軟體漏洞重現)：Mythos **83.1%** vs. Opus 4.6 (66.6%)
- **真實世界的漏洞挖掘能力**：
  - **名詞科普：什麼是「零日漏洞 (Zero-day)」？**
    - 簡單解釋：「原廠自己都不知道、也還沒辦法修補的致命坑洞」。因為連一天（Zero days）的時間都沒得補，駭客利用這種漏洞攻擊是完全防不勝防的。
  - 總戰績：成功揪出**數千個零日漏洞**，遍佈每個主要 OS 與瀏覽器。
  - 對比：前代 Opus 4.6 大約找到 500 個。
- **三個恐怖的具體「陳年漏洞」案例**：
  1. OpenBSD 漏洞：潛伏 **27 年**，可遭遠端癱瘓系統。
  2. FFmpeg 漏洞：存在 **16 年**，被自動化工具掃過 500 萬次都沒發現。
  3. Linux 核心漏洞鏈：可被串聯利用並取得 root 最高權限。

### [Part 3] 防禦策略：武裝防守方 — Project Glasswing（約 3 分鐘）
- **防禦計畫核心理念**：不急著賣錢變現，而是先發放頂級武器給防禦者補洞，爭取修補的安全空窗期。
- **存取名單極度受限**：模型僅提供給 12 家核心合作夥伴（共約 40 間組織）。
- **全明星護城河陣容（代表企業）**：
  - Cloud / Tech: AWS, Apple, Google, Microsoft, NVIDIA, Broadcom
  - Cybersecurity: CrowdStrike, Palo Alto Networks, Cisco
  - 其他巨頭: JPMorganChase, Linux Foundation
- **資源投入**：
  - **1 億美元**：投入供防禦方使用的模型額度。
  - **400 萬美元**：直接贊助給開源資安組織。

### [Part 4] 專家視角與結語反思（約 2 分鐘）
- **各界專家呼應（供選擇引用）**：
  - **Igor Tsyganskiy (Microsoft Global CISO)**：AI 找漏洞的能力已與專業人類並駕齊驅。
  - **Jim Zemlin (Linux 基金會 CEO)**：Linux 是網路核心，需要有頂尖 AI 幫忙抓出最底層的蟲。
  - **Palo Alto Networks / Cisco 代表觀點**：底層漏洞會引發全球連鎖災難，目前防禦方是在「搶修補的時間差」。
- **探討與反思 (拋給聽眾的結語問題)**：
  - Anthropic 這種「限用以武裝防禦者」的作法，反映了對社會負責的態度。
  - **未來的隱憂**：如果下個月競爭對手推出同樣強大的開源模型並放到網路上，這段「防禦優先」的時間差會不會瞬間歸零？
  - 拋出互動思考點：對於這種不直接發售、僅限頭部資安大廠使用的政策，你覺得是明智的保護，還是變相壟斷？
