## 背景與技術定位

我畢業於國立暨南國際大學資訊管理研究所，過去十多年持續在電商平台、企業內部系統與物流領域累積實務經驗。技術棧以 JavaScript、TypeScript、React、Next.js、React Native、C# 與 .NET 為主。

在 Newegg 的職涯中，我參與過 B2B 與 B2C 電商平台、Mobile Site 與 React Native App 的開發與維護，也在後期逐步承擔 Frontend Leader / Staff Engineer 的角色，協調跨台灣、美國、中國團隊的需求與技術決策。

近年的一段經驗，則是在昭津國際物流負責新一代物流與貨代管理系統的前端架構與開發。這個專案讓我同時處理從 0 到 1 的 React / Next.js 工程架構、多語系平台、Azure CI/CD，以及與既有 OutSystems TMS / 倉儲系統的整合。

## 我正在做的事情

我最有興趣的工作，通常落在下面幾種交集：

- **Enterprise Frontend Architecture**：讓前端可以長期擴展、可維護、可安全地協作。
- **System Design for Real Operations**：系統不只是畫面，而是要支撐真實的物流、電商與企業流程。
- **AI-assisted Engineering**：把 Copilot、Codex、Claude、ChatGPT、MCP、skill / instruction 等方法整合到真實的工程流程裡。

我不太把 AI 當作單次問答工具，而是更重視怎麼讓專案上下文、開發規範與文件結構更容易被 AI 理解與延伸。

## 我的工作方式

我偏好把問題工程化，而不是只追求功能「先能動」。

對我來說，一個健康的系統通常需要：

- 清楚的模組邊界與資料流責任
- 可讀、可維護、可逐步重構的專案結構
- 能支撐多人協作與 code review 的規範
- 文件與 instruction 足夠清楚，讓新成員或 AI agent 都能快速接手

這也是為什麼我對前端架構、工程效率與技術文件寫作一直保持高度投入。

## 我目前關注的方向

我目前持續投入在 **AI-assisted software engineering**。

我相信未來工程師的價值，不只是寫出程式，而是能夠設計更有效率的開發流程、建立可被 AI 理解的專案上下文，並把 AI 真正整合進工程體系，而不是停留在補全與問答層。
